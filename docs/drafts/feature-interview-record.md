좋습니다. 이 기능은 단순히 “전사 결과를 보는 기능”이 아니라,
**실전 면접 음성과 스크립트, 질문 카드, 복기를 하나의 시간축으로 연결하는 인터뷰 리플레이 기능**으로 설계하는 것이 맞습니다.

핵심은 아래 3가지입니다.

* **전체 인터뷰 세션에 음원 플레이어 추가**
* **각 질문 카드에도 해당 구간 재생 플레이어 추가**
* **전사 스크립트에 타임스탬프를 붙이고, 클릭 시 해당 시점으로 이동 재생**

이렇게 되면 사용자는 실전 면접을 “문서”로만 보는 것이 아니라,
**실제 음성 흐름까지 포함해 정확하게 복기**할 수 있습니다.

# 1. 기능 한 줄 정의

**실전 면접 음성 파일 업로드 시 인터뷰 전체 세션과 각 질문 카드에 구간 재생 가능한 음원 플레이어를 제공하고, 전사 스크립트 및 질문/답변에 타임스탬프를 연결해 클릭만으로 해당 시점으로 이동 재생할 수 있게 하는 기능**

### 추천 기능명

* **타임라인 복기 플레이어**

**인터뷰 리플레이**

# 3. 왜 이 기능이 중요한지

실전 면접 복기에서 텍스트만 보면 놓치는 것이 많습니다.

예를 들면 아래와 같습니다.

* 어디서 머뭇거렸는지
* 질문 의도를 제대로 이해하지 못한 순간이 언제였는지
* 답변 톤이 흔들린 구간이 어디였는지
* 면접관이 다시 파고든 시점이 정확히 언제였는지
* 같은 질문이라도 실제 말한 방식이 어땠는지

그래서 이 기능의 핵심 가치는 단순 재생이 아니라 아래입니다.

### 핵심 가치

* 스크립트와 실제 음성을 **정확히 연결**
* 질문 카드와 실제 답변 구간을 **즉시 검증**
* 복기 속도 향상
* 질문 단위 재연습 품질 향상
* 실제 면접 흐름을 시간축 기반으로 학습 가능

즉,
**“읽는 복기”를 “듣고 확인하는 복기”로 확장하는 기능**입니다.

# 4. 핵심 기능 구성

## A. 전체 인터뷰 세션 플레이어

실전 면접 1회에 대해 상단 또는 고정 영역에 전체 음원 플레이어를 둡니다.

### 제공 기능

* 재생 / 일시정지
* 현재 시간 / 전체 길이
* 5초, 10초 단위 이동
* 배속 조절
* 진행 바 탐색
* 현재 재생 중 구간 하이라이트
* 선택 질문 위치로 즉시 이동

이 플레이어는 해당 인터뷰 세션 전체의 기준 플레이어가 됩니다.

---

## B. 질문 카드 구간 플레이어

각 질문 카드에도 독립적으로 보이는 “구간 재생” 버튼 또는 미니 플레이어를 제공합니다.

### 동작 방식

질문 카드에는 보통 아래 구간이 연결됩니다.

* 질문 시작 시점
* 질문 종료 시점
* 답변 시작 시점
* 답변 종료 시점

이 중 어떤 범위를 재생할지 옵션을 둘 수 있습니다.

### 재생 모드 예시

* **질문만 듣기**
* **답변만 듣기**
* **질문+답변 전체 듣기**
* **해당 질문에서 이어진 꼬리질문까지 듣기**

초기에는 아래 두 가지만 있어도 충분합니다.

* 질문 구간 재생
* 질문+답변 구간 재생

---

## C. 타임스탬프가 포함된 스크립트

음성 전사 결과는 반드시 시간 정보와 함께 저장되어야 합니다.

### 스크립트 표시 예시

* `[00:12]` 자기소개 부탁드립니다.
* `[00:18]` 안녕하세요. 저는 백엔드 개발자로...
* `[01:42]` Redis를 사용하셨다고 했는데...
* `[01:49]` 네, 당시 응답 속도 병목이...

이 타임스탬프는 단순 텍스트가 아니라 **클릭 가능한 링크형 인터랙션**이어야 합니다.

---

## D. 클릭 가능한 타임스탬프

유튜브처럼 타임스탬프를 클릭하면 플레이어가 해당 위치로 이동해야 합니다.

### 기대 동작

* 사용자가 `[01:42]` 클릭
* 플레이어가 1분 42초 지점으로 seek
* 자동으로 재생 또는 대기 상태 진입
* 해당 문장/질문 카드가 active 상태로 강조

### 추천 동작 정책

기본값은 아래가 가장 자연스럽습니다.

* **클릭 시 해당 시점으로 이동 + 자동 재생**
* 설정으로 “클릭 시 이동만” 옵션 제공 가능

# 5. 적용 대상

이 기능은 아래 두 레벨에서 반드시 동작해야 합니다.

## 1) 인터뷰 세션 레벨

실전 면접 전체 화면에서

* 전체 플레이어
* 전체 스크립트
* 전체 질문 흐름
* 타임라인 이동

## 2) 질문 카드 레벨

질문 상세 카드나 리스트에서

* 해당 질문 구간 재생
* 해당 질문 시작 타임스탬프 표시
* 답변 시작 타임스탬프 표시
* 플레이어 연동 이동

이 둘이 자연스럽게 연결되어야 합니다.

# 6. 사용자 시나리오

## 시나리오 1: 전체 복기

1. 사용자가 면접 음성 파일을 업로드합니다.
2. 앱이 전사와 타임스탬프를 생성합니다.
3. 인터뷰 상세 화면에서 상단 플레이어와 전체 스크립트를 확인합니다.
4. 특정 타임스탬프를 클릭하면 해당 위치로 이동 재생합니다.
5. 재생 중인 문장과 질문 카드가 함께 강조됩니다.

---

## 시나리오 2: 질문 카드 중심 복기

1. 사용자가 질문 카드 리스트를 봅니다.
2. 특정 질문 카드에서 “질문+답변 듣기”를 누릅니다.
3. 플레이어가 해당 구간으로 이동해 바로 재생합니다.
4. 사용자는 실제 답변 톤, 머뭇거림, 길이를 확인합니다.
5. 같은 카드에서 “다시 연습”으로 이어집니다.

---

## 시나리오 3: 스크립트 검수

1. 전사 결과에서 오인식이 의심되는 문장을 확인합니다.
2. 해당 문장의 타임스탬프를 클릭합니다.
3. 실제 음성을 바로 재생해 듣습니다.
4. 사용자가 정제본을 수정합니다.

이 시나리오는 전사 품질 검증에 매우 중요합니다.

# 8. 타임스탬프 단위 설계

타임스탬프는 최소 3단계 단위로 저장하는 것이 좋습니다.

## 1) 세그먼트 단위

전사 시스템이 뽑은 문장 또는 발화 단위

예:

* startMs
* endMs
* speaker
* text

## 2) 질문 단위

질문 전체 시작/종료

예:

* questionStartMs
* questionEndMs

## 3) 답변 단위

답변 전체 시작/종료

예:

* answerStartMs
* answerEndMs

이렇게 해야

* 스크립트 이동
* 질문 카드 재생
* 구간 반복 재생
  이 모두 가능합니다.

# 9. 플레이어 인터랙션 설계

## 기본 원칙

**플레이어는 여러 개가 아니라, 실제 재생 엔진은 하나이고 UI는 여러 곳에서 제어하는 구조**가 좋습니다.

즉,

* 상단 전체 플레이어
* 질문 카드 미니 플레이어
* 스크립트 타임스탬프

이 세 곳이 모두 같은 플레이어 상태를 제어해야 합니다.

이유는 다음과 같습니다.

* 동시에 여러 오디오가 재생되면 UX가 나쁩니다.
* 현재 어떤 지점이 재생 중인지 상태 동기화가 쉬워집니다.
* 구현도 더 안정적입니다.

---

## 추천 플레이어 상태

* currentTimeMs
* durationMs
* isPlaying
* playbackRate
* activeSegmentId
* activeQuestionId
* selectedRangeStartMs
* selectedRangeEndMs
* repeatMode

---

## 구간 재생

질문 카드에서 “질문+답변 듣기”를 누르면 아래처럼 동작합니다.

1. 전체 플레이어가 해당 구간 시작으로 이동
2. 구간 범위를 active range로 설정
3. 재생 시작
4. 구간 종료 시 자동 정지 또는 반복

### 구간 종료 정책

MVP에서는 아래가 적절합니다.

* 기본: 구간 끝에서 자동 정지
* 옵션: 구간 반복 재생

# 11. 전사 파이프라인 설계

실전 면접 음성 파일을 업로드하면 아래 순서로 처리하는 것이 좋습니다.

## 1단계: 음원 저장 및 메타 추출

* 파일 업로드
* 재생 길이 추출
* 포맷 검사
* 필요 시 오디오 정규화

## 2단계: 전사

* 세그먼트별 타임스탬프 생성
* 화자 분리 시도
* 원문 전사 생성

## 3단계: 정제

* 기술 용어 보정
* 문맥상 이상한 단어 보정
* 정제본 생성

## 4단계: 질문/답변 구조화

* 질문 구간
* 답변 구간
* follow-up 관계
* 질문 카드 생성

## 5단계: 플레이어 메타 생성

* waveform 데이터
* 스크립트-타임스탬프 맵
* 질문 카드-오디오 범위 맵

# 12. 시각적 UX 강화 포인트

## A. 현재 재생 중 문장 하이라이트

오디오가 재생되면 현재 재생 구간의 스크립트 줄을 자동 강조합니다.

## B. 현재 재생 중 질문 카드 active 표시

질문 카드 목록에서도 현재 듣고 있는 질문 카드가 활성화됩니다.

## C. 타임라인 미니맵

상단 플레이어 아래에 전체 인터뷰 타임라인을 두고,

* 질문 구간
* 꼬리질문 구간
* 약했던 답변 구간
  을 작은 마커로 표시할 수 있습니다.

이건 2차 확장으로 매우 좋습니다.

## D. waveform 표시

가능하다면 파형을 같이 보여주면 재생 UX가 좋아집니다.
다만 초기 MVP에서는 필수는 아닙니다.

# 13. 질문 카드의 오디오 연동 방식

질문 카드에는 단순 “재생 버튼”보다 아래 구조가 좋습니다.

### 카드 예시

* 질문 텍스트
* `[01:42] 질문`
* `[01:49] 답변`
* `질문 듣기`
* `질문+답변 듣기`
* `스크립트로 이동`

### 추가 가능 기능

* 이 구간 반복
* 북마크
* 코멘트 달기
* 약점 태그 연결

이렇게 되면 질문 카드는 단순 텍스트 객체가 아니라
**실제 음성과 연결된 학습 단위**가 됩니다.

# 14. 유튜브식 타임스탬프 UX 기준

유튜브와 비슷하게 만들고자 할 때 중요한 규칙은 아래입니다.

## 필수 동작

* `[mm:ss]` 또는 `[hh:mm:ss]` 형식 표시
* 클릭 가능
* 클릭 시 정확한 시점으로 seek
* 플레이어 상태와 동기화
* 현재 재생 위치가 바뀌면 해당 타임스탬프가 활성화

## 있으면 좋은 동작

* hover 시 “이 구간 재생” 툴팁
* 우클릭 또는 더보기 메뉴에서

    * 질문 구간 재생
    * 답변 구간 재생
    * 링크 복사
* 모바일에서는 탭 시 바로 이동 재생

---

# 15. 이 기능이 기존 기능과 연결되는 방식

## A. 스크립트 정제와 연결

오인식 단어가 있으면 그 타임스탬프로 바로 이동해 확인 후 수정할 수 있습니다.

## B. 질문 카드와 연결

질문 카드가 실제 음성 구간과 연결되므로 복기 품질이 올라갑니다.

## C. 실전 면접 복기와 연결

“이 질문에서 내가 왜 약했는지”를 텍스트뿐 아니라 실제 답변 톤으로도 확인할 수 있습니다.

## D. 시뮬레이션 기능과 연결

이전 실전 질문을 재생해 들은 뒤 동일 스타일 시뮬레이션으로 바로 들어갈 수 있습니다.

---

# 16. 예시 사용 흐름

## 인터뷰 상세

* 상단 플레이어
* 전체 스크립트
* 질문 카드 리스트

## 사용 행동

* 사용자가 질문 카드 클릭
* 플레이어가 해당 시점으로 이동
* 스크립트도 해당 구간으로 스크롤
* 해당 질문/답변 줄 하이라이트
* 재생 시작

## 다시 연습

* 사용자가 그 카드에서 “다시 연습”
* 동일 질문으로 시뮬레이션 시작

이 흐름이 가장 자연스럽습니다.

# 17. MVP 범위 제안

## MVP 포함

* 음원 업로드
* 전체 인터뷰 플레이어
* 세그먼트 타임스탬프 포함 전사
* 타임스탬프 클릭 시 seek
* 질문 카드별 시작/종료 시점 저장
* 질문 카드에서 구간 재생
* 현재 재생 중 스크립트 하이라이트

## 2차 확장

* 질문+답변 구간 반복 재생
* 파형 표시
* 재생 위치에 따른 자동 스크롤
* active question 동기화
* 꼬리질문 구간 묶음 재생

## 3차 확장

* 북마크 구간 저장
* 오디오 클립 내보내기
* 코멘트와 오디오 구간 직접 연결
* 비교 청취

    * 실제 답변 듣기
    * 새 연습 답변 듣기

# 19. 최종 기능 정의

## 기능 요약

**실전 면접 음성 파일을 업로드하면 인터뷰 전체 세션과 각 질문 카드에 음원 플레이어를 제공하고, 전사 스크립트 및 질문/답변에 타임스탬프를 연결하여 유튜브처럼 클릭 시 해당 시간으로 이동 재생할 수 있게 하는 기능**

## 핵심 가치

* 텍스트와 실제 음성을 정밀하게 연결
* 질문 단위 복기 정확도 향상
* 전사 검수 효율 개선
* 실제 답변 톤과 흐름까지 포함한 학습 가능

## 우선순위

1. 세그먼트 타임스탬프 생성
2. 전체 플레이어
3. 질문 카드 구간 재생
4. 클릭형 타임스탬프 seek
5. active 상태 동기화
6. 반복 재생/파형 등 고급 UX

원하시면 이어서 이 기능까지 포함해서
**실전 면접 복기 기능 전체 PRD** 형태로 더 구체적으로 정리해드리겠습니다.
